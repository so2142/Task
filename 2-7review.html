>> git branch [ ] プロジェクトを分岐させることで、プロジェクト本体に影響を与えずに開発することができる プロジェクトを同時進行で行う際に使用し、不具合が起きた際も個々で対処可能となる >> git pull ブランチを切って作業を行なっている途中で、開発環境が変わることがある。その際に、リモートリポジトリに上がった変更された環境を、ローカルリポジトリに取り込むこと。 データのダウンロードのようなもの。 >> 同じファイル名の同じ場所（行）に同時に変更を行なってしまった時に起こる衝突のこと。 たとえば、開発環境が更新された後、同時に進めていたそれぞれのブランチの同じ行で変更が行われることがある。 その時に、同じファイル名の同じ場所に変更が起こったとしてバージョンの衝突が起こる。これを指す。 この場合、先に開発環境が変更された方を優先し、後に変更されたほうを次行に書く。 >> git merge [ ] 加えた修正を統合する作業のこと。 変更を加えたブランチを元のブランチにマージして修正、反映をする。 >> git pull-request GitHubの機能の一つで、ローカルリポジトリでの変更を責任者にプルしてもらえるようにお願いすること。 >> git fork GitHubが提供する機能の一つで、開発元を援助する前提で、開発元のリポジトリをGitHub上でクローンできる クローンしたリポジトリは独立するため、交わることはない >> git stash ブランチを切って作業している途中で、別の作業をやることになった場合等、一時的に中断せざるを得ない時、 または、作業するブランチを間違えたと気づいた時に、コミットしていない変更を一時退避させること。 >> git diff コミット同士の差分の前後を比較、確認できる機能のこと。
